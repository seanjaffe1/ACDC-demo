\section{Background}
\label{sec:background}

\subsection{Statistical Shape Models}
\label{sec:bg_ssm}

Statistical shape models (SSMs) represent shape variation within a
population by applying principal component analysis (PCA) to a set of
registered point correspondences~\cite{cootes1995active}.
Given $N$ shapes with $V$ vertices each, we flatten each shape to a
vector $\mathbf{s}_i \in \R^{3V}$ and compute the empirical covariance.
New shapes are sampled as
\begin{equation}
    \hat{\mathbf{s}} = \bar{\mathbf{s}} + \mathbf{P}\mathbf{b},
    \quad \mathbf{b} \sim \mathcal{N}(\mathbf{0}, \boldsymbol{\Lambda}),
\end{equation}
where $\bar{\mathbf{s}}$ is the mean shape, $\mathbf{P}$ are the
principal modes, and $\boldsymbol{\Lambda}$ is the diagonal matrix of
eigenvalues.
SSMs are computationally efficient but capture only linear variation,
failing to represent non-linear deformation modes such as the asymmetric
remodelling seen in hypertrophic cardiomyopathy.

\subsection{Geometric Deep Learning and SE(3) Equivariance}
\label{sec:bg_geo}

A function $f$ is \emph{equivariant} under a symmetry group $G$ if
\begin{equation}
    f(g \cdot \mathbf{x}) = g \cdot f(\mathbf{x})
    \quad \forall\, g \in G,\; \mathbf{x} \in \mathcal{X}.
\end{equation}
For 3-D point sets, the relevant group is $\SE = \SO \ltimes \R^3$,
the group of rotations and translations in three dimensions.
A neural network operating on point positions that is equivariant to
$\SE$ will produce geometrically consistent outputs regardless of the
coordinate frame in which the input is expressed.

We use the \emph{$E(n)$ Equivariant Graph Neural Network} (EGNN)
of~\citet{satorras2021n} as our equivariant backbone.
Each EGNN layer jointly updates scalar node features $\mathbf{h}_i$
and equivariant coordinates $\mathbf{x}_i$:
\begin{align}
    \mathbf{m}_{ij} &= \phi_e\!\left(\mathbf{h}_i,\, \mathbf{h}_j,\,
                       \norm{\mathbf{x}_i - \mathbf{x}_j}^2\right),
    \label{eq:egnn_msg} \\
    \mathbf{x}_i    &\leftarrow \mathbf{x}_i
                       + \sum_{j \neq i}
                         (\mathbf{x}_i - \mathbf{x}_j)\,
                         \phi_x(\mathbf{m}_{ij}),
    \label{eq:egnn_coord} \\
    \mathbf{h}_i    &\leftarrow \phi_h\!\left(\mathbf{h}_i,\,
                       \sum_{j}\mathbf{m}_{ij}\right),
    \label{eq:egnn_node}
\end{align}
where $\phi_e$, $\phi_x$, and $\phi_h$ are multi-layer perceptrons.
Because the coordinate update in~\eqref{eq:egnn_coord} depends only on
relative positions and the scalar $\phi_x$, the overall mapping
$(\mathbf{H}, \mathbf{X}) \mapsto (\mathbf{H}', \mathbf{X}')$ is
equivariant to $\SE$ by construction.

\subsection{Diffusion Models and Flow Matching}
\label{sec:bg_flow}

Score-based diffusion models~\cite{song2020score} learn to reverse a
forward noising process, but typically require thousands of denoising
steps at inference time.
\emph{Flow matching}~\cite{lipman2022flow} instead defines a
deterministic probability path between a noise distribution $p_0$ and
the data distribution $p_1$, parameterising the corresponding velocity
field $v_\theta(\mathbf{x}_t, t)$ that transports samples from $p_0$
to $p_1$.

Under the \emph{Optimal Transport} (OT) path~\cite{albergo2023building},
interpolating trajectories are straight lines:
\begin{equation}
    \mathbf{x}_t = (1 - t)\,\mathbf{x}_0 + t\,\mathbf{x}_1,
    \quad t \sim \mathcal{U}[0, 1],
\end{equation}
yielding the simple regression objective
\begin{equation}
    \mathcal{L}_\text{CFM}(\theta)
    = \mathbb{E}_{t,\, \mathbf{x}_0 \sim p_0,\, \mathbf{x}_1 \sim p_1}
      \left[ \norm{v_\theta(\mathbf{x}_t, t, \mathbf{z})
             - (\mathbf{x}_1 - \mathbf{x}_0)}^2 \right],
    \label{eq:cfm_loss}
\end{equation}
where $\mathbf{z}$ is an optional conditioning variable.
Samples are generated by integrating $v_\theta$ from $t=0$ to $t=1$
using a standard ODE solver (Euler or RK4).
Compared to score-based methods, flow matching is faster to train
(single forward pass per sample) and requires far fewer integration
steps at inference.

\section{Method}
\label{sec:method}

\subsection{Data and Preprocessing}
\label{sec:method_data}

We use the \emph{Automated Cardiac Diagnosis Challenge} (ACDC)
dataset~\cite{bernard2018deep}, which provides cine-MRI volumes with
expert segmentation masks for 100 subjects spanning five pathology
classes: normal (NOR), myocardial infarction (MINF), dilated
cardiomyopathy (DCM), hypertrophic cardiomyopathy (HCM), and right
ventricular abnormality (RV).
Segmentations label the left ventricle (LV, label~3), right ventricle
(RV cavity, label~1), and myocardium (Myo, label~2) at end-diastole
(ED) and end-systole (ES).

\paragraph{Mesh extraction.}
For each structure at each frame, we apply the marching cubes
algorithm~\cite{lorensen1987marching} to the binary segmentation mask,
using the physical voxel spacing as the step size to produce meshes in
millimetre coordinates.
We then apply ten iterations of Laplacian smoothing to reduce
staircase artefacts, and decimate each mesh to approximately $2{,}500$
vertices via quadric edge collapse, targeting a balance between
geometric fidelity and computational cost.

\paragraph{Template registration.}
To establish vertex correspondence across subjects we select a single
reference subject and register all others to it.
Registration proceeds in two stages: (i)~a coarse Procrustes
alignment using the Kabsch algorithm, and (ii)~refinement with
Iterative Closest Point (ICP).
After alignment, each mesh is centred at the origin by subtracting
its centroid, removing translation as a degree of freedom.

\paragraph{Graph construction.}
Each registered mesh is stored as a PyTorch Geometric
\texttt{Data} object containing vertex positions
$\mathbf{X} \in \R^{V \times 3}$, a $k$-nearest-neighbour edge index
($k = 8$) computed in the registered coordinate frame, the face
connectivity for surface-area and volume calculations, and an integer
pathology label $y \in \{0, \ldots, 4\}$.

\subsection{Model Architecture}
\label{sec:method_arch}

Our model consists of two learned components: a graph encoder that maps
a cardiac mesh to a latent code, and a flow-matching velocity field
conditioned on that code.
Figure~\ref{fig:architecture} provides an overview.

\paragraph{EGNN Encoder.}
The encoder is a four-layer EGNN followed by mean-and-max global
pooling, producing a Gaussian posterior
$q_\phi(\mathbf{z} \mid \mathcal{G}) = \mathcal{N}(\boldsymbol{\mu},
\text{diag}(\boldsymbol{\sigma}^2))$
with $\boldsymbol{\mu}, \log\boldsymbol{\sigma} \in \R^{64}$.
Initial node features are set to a constant vector
(no semantic node attributes are available), so all information must be
inferred from the geometric structure of the graph.

\paragraph{Equivariant Velocity Field.}
The velocity field $v_\theta(\mathbf{x}_t, t, \mathbf{z})$ is a
four-layer EGNN where each node's input feature is the concatenation
of a sinusoidal time embedding and the conditioning code $\mathbf{z}$
projected to the hidden dimension.
The network outputs a velocity vector in $\R^3$ for each vertex.
Because position updates in EGNN depend only on relative coordinates,
the velocity field inherits $\SE$-equivariance: rotating the input
mesh rotates the predicted velocities by the same rotation.

\paragraph{Training Objective.}
The full objective is
\begin{equation}
    \mathcal{L} = \mathcal{L}_\text{CFM}(\theta)
                + \beta\, D_\text{KL}\!\left(
                    q_\phi(\mathbf{z} \mid \mathcal{G})
                    \;\|\;
                    \mathcal{N}(\mathbf{0}, \mathbf{I})
                  \right),
    \label{eq:total_loss}
\end{equation}
where $\mathcal{L}_\text{CFM}$ is the OT flow matching loss from
Equation~\eqref{eq:cfm_loss} and $\beta = 10^{-3}$.

\subsection{Baselines}
\label{sec:method_baselines}

We compare against four baselines, each designed to ablate a specific
modelling choice.

\textbf{Baseline~1: PCA Shape Model.}
Classical linear SSM fit to flattened vertex positions.
Justifies the use of deep learning at all.

\textbf{Baseline~2: Voxel Diffusion.}
OT flow matching applied to binary segmentation volumes on a $32^3$
grid, using a 3-D U-Net velocity field.
Justifies the mesh representation over voxel grids.

\textbf{Baseline~3: PointNet++ VAE.}
PointNet++ encoder with a FoldingNet decoder trained as a VAE
(no diffusion/flow).
Justifies the distributional modelling component.

\textbf{Baseline~4: Non-equivariant Flow Matching.}
Identical architecture to our main model, but EGNN layers replaced by
standard MLP message passing; random rotation augmentation is applied
during training to compensate.
This is the \emph{critical} ablation: it directly validates the
geometric deep learning contribution.
